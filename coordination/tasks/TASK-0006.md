## Contexto

El pipeline del Runner avanza como **diseño/revisión**, con el Runner siempre **bloqueado**:

- `TASK-0004` (Runner Design Review, PR #252) definió el Runner como **coordinador mecánico solo-lectura**, nunca autoridad.
- `TASK-0005` (Runner Manual Dry-Run, PR #253) simuló **a mano** (sin código) el protocolo sobre `main`: confirmó que es operable sin autoridad y produjo cuatro fricciones (**F1–F4**) que deben alimentar el MVP.

El siguiente paso recomendado por el dry-run es convertir ese aprendizaje en una **especificación técnica mínima** del MVP más pequeño posible: el **Read-Only Queue Reporter**. Este PR es esa especificación; **no** implementa nada.

## Objetivo

Especificar el MVP Read-Only Queue Reporter como un **lector manual** que en una implementación futura solo: lee `coordination/tasks/*.md` desde `main`, extrae front-matter, detecta artefactos presentes, clasifica cada tarea, imprime/reporta una cola y marca ambigüedades — **sin** escribir en el repo, **sin** ejecutar acciones y **sin** decidir autoridad. El diseño debe incorporar F1–F4 como requisitos explícitos.

## Alcance y límites

- **Solo `coordination/`**, y solo: `tasks/TASK-0006.md`, `proposals/READ_ONLY_QUEUE_REPORTER_MVP.md`, `proposals/READ_ONLY_QUEUE_REPORTER_ACCEPTANCE_TESTS.md`.
- Trabajo en rama `coordination/task-0006-read-only-queue-reporter-proposal`, nunca `main`.
- Fuentes obligatorias referenciadas: `RUNNER_DESIGN_REVIEW.md`, `RUNNER_MANUAL_PROTOCOL_DRAFT.md`, `RUNNER_DRY_RUN_OBSERVATIONS.md`, `worklogs/TASK-0005.WORKLOG.md`, `reports/TASK-0005.FINAL_REPORT.md`, `STATE.md`, `README.md`, `RULES_OF_ENGAGEMENT.md`, `AGENT_CONTRACT.md`, `schemas/`.

- **No** implementar MVP, scripts, CLI, GitHub Actions, cron, Docker, túnel/VPS, headless automation, tokens/secretos, notificaciones reales.
- **No** escribir estados; **no** fijar `HANDOFF_TO_MSO`; **no** mover ninguna tarea.
- **No** tocar `STATE.md`, `TASK-0002.md`, `TASK-0004.md`, `TASK-0005.md`, `TASK-0002.DECISION.md`, `TASK-0002.DECISION_CANDIDATE.md`, MSO/Police/Policy/Auth, ni `cowork/`.
